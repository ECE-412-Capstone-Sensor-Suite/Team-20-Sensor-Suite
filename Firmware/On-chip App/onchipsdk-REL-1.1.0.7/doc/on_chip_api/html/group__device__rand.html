Random number generator device.

Note
Use the Device API to control this device. This page details the parameters to pass to the functions of that API.
The On-Chip Firmware Development Guide contains code snippets and sample applications, some of which use this device.

The following table indicates which Device API functions apply to this device. Calling one of the unsupported functions on this device returns DN_ERR_NOT_SUPPORTED. Only the supported functions are detailed below.

dn_open dn_ioctl dn_read dn_write dn_close
YES NO YES NO YES

Overview

The DN_RAND_DEV_ID device samples the thermal noise of an internal circuit to generate a sequence of random bytes. Your application can use this device when it boots to generate a seed for a pseudo-random number generator.

Warning
While the Random Number Generator (RNG) is believed to be a high quality source of random data, the mote's RNG is not intended to be used as a source of cryptographic quality random numbers, and the application should not use the RNG for cryptographic operations.
Note
You cannot use this device after having issuing the join or search commands to the mote. Doing otherwise will result in API returning DN_ERR_STATE error code. Also the device can't be used if the mote is configured to join automatically.

Open the device

To open this device, call dn_open() with the following parameters:

Parameter Name Parameter Value
device DN_RAND_DEV_ID
args NULL
argLen 0
Note
This function is provided so all devices have the same interface. This function, however, has no effect on the state of the device. This means that you can use this device without calling this function. This also means that you cannot use the return value of this function to decide whether another thread has already opened this device.

Read from the device

To generate a sequence of random bytes, start by allocation a buffer of size the number of random bytes you need.

Then, call the dn_read() function with the following parameters:

Parameter Name Parameter Value
device DN_RAND_DEV_ID
buf Points to the buffer you allocated.
bufSize The number in bytes of the buffer you allocated.
Postcondition
After this function returns, the buffer you allocated contains a random sequence of bytes.
Note
This function will return DN_ERR_STATE if you call it after calling dnm_loc_joinCmd().

Close the device

Close the device by calling the dn_close() function.

Note
This function is provided so all devices have the same interface. This function, however, has no effect on the state of the device. This means that you can use use this device even after calling this function.
