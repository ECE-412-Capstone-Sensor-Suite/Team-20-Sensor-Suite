Description

Direct access to the flash memory.

The On-Chip Firmware Development Guide covers how the flash is organized inside the LTC5800 chip, and how you can alter the partition table to create an extra "raw" flash partition, i.e. a unformatted sequence of bytes.

You can use the functions defined in this component to erase, read and write that flash partition.

Warning
Due to the nature of flash memory, your application can not write a given word in flash memory more than once between erases.

INT32U  offset,
INT8U *  data,
INT32U  len 
)

Write data to the specified partition.

Warning
Make sure that the value passed in the offset is word-aligned
Due to the nature of flash memory, your application cannot write a given word in flash memory more than once between erases.
Parameters
[in]parDescriptor of the partition to write to.
[in]offsetOffset in bytes starting from the beginning of the partition to start writing to.
[in]dataPointer to a buffer containing the data to be written to the partition.
[in]lenNumber of bytes from data buffer to write to the partition
Returns
DN_ERR_NONE if the function completes successfully.
DN_ERR_INVALID if the partition descriptor is invalid.
DN_ERR_INVALID if the offset is invalid.
DN_ERR_WRITE if writing to flash failed.
DN_ERR_RDONLY if the partition is mounted read-only.
DN_ERR_SIZE if you try to write too many bytes to the partition.
