Description

Serial Peripheral Interface (SPI) digital bus device.

Note
Use the Device API to control this device. This page details the parameters to pass to the functions of that API.
The On-Chip Firmware Development Guide contains code snippets and sample applications, some of which use this device.

The following table indicates which Device API functions apply to this device. Calling one of the unsupported functions on this device returns DN_ERR_NOT_SUPPORTED. Only the supported functions are detailed below.

dn_open dn_ioctl dn_read dn_write dn_close
YES YES NO NO YES*

Open the device

You need to call dn_open() to open this device before using any of the other functions. To open this device, use the dn_open() function with the following parameters:

Parameter Name Parameter Value
device DN_SPI_DEV_ID
args Points to a pre-populated dn_spi_open_args_t structure.
argLen Size of dn_spi_open_args_t.

Control the device

Precondition
Before using this function, make sure to open the device using dn_open()

Communication over SPI is synchronous: for each byte the master sends to the slave, the slave sends a byte to the master. Transmission and reception happen at the same time. For that reason, the tranfer operation is done using the Device API's dn_ioctl() function rather than dn_read() or dn_write().

To send and receive data to/from a slave device, use the dn_ioctl() function with the following parameters:

Parameter Name Parameter Value
device DN_SPI_DEV_ID
request DN_IOCTL_SPI_TRANSFER
args Points to a pre-populated dn_ioctl_spi_transfer_t structure.
argLen Size of dn_ioctl_spi_transfer_t.

This triggers a (possibly periodic) transfer of data, as specified in the args parameter. Call this function again for subsequent transfers.

The dn_ioctl_spi_transfer_t structure defines the clock speed, phase, polarity and chip select line to use, among other SPI settings. This allows your application to control those settings with a per-transfer granularity.

Close the device

Warning
You cannot reopen the SPI device after closing it. We recommend you keep the SPI device open as long as you use it.

Close the SPI device by calling the dn_close() function.
