Description

A file-like interface to access all the mote's devices.

The SmartMesh library allows your application to access the LTC5800's devices. Access to all devices, including sensors, actuators, ADC and digital interfaces is done through the set of functions listed in this page.

This interface exposes the devices as if they were files. That is, you open a device, write to it, read from it, and close the device when you're done. The SmartMesh library turns these high-level commands into the appropriate low-level hardware settings, while keeping the full functionality by allowing you to fine-tune the operation of the device.

int dn_open ( int  device,
const void *  args,
int  argLen 
)

Open a device.

Your application must open a device before using it (i.e. calling dn_ioctl, dn_read or dn_write on that device). You must never open a device twice.

Postcondition
After this function returns successfully, the device is open and ready to be used.
Parameters
[in]deviceThe device you want to open. Acceptable value are listed in dn_device_t.
[in]argsPointer to the arguments for opening this device. What this points to depends on the type of device you are about to open; refer to the documentation of that device. If no arguments are expected, set to NULL.
[in]argLenLength of the structure pointed to by the args parameter. The value to specify here depends on the type of device you are about to open; refer to the documentation of that device. If no arguments are expected, set to 0.
Returns
DN_ERR_NONE when the device was successfully opened.
DN_ERR_INVALID if the device specified does not exist.
DN_ERR_NOT_SUPPORTED if the device specified exists but does not support to be opened.
This function can also return device-specific error codes. These are listed in device documentation. Value will always be taken from dn_error_t.
int dn_ioctl ( int  device,
int  request,
const void *  args,
int  argLen 
)

Control a previously opened device.

Precondition
You must have opened the device before calling this function.
Postcondition
Depending on the device and type of request, the state of the device might have changed after this function returns successfully. Refer to the device-specific documentation for details.

"Controlling" a device is a generic term which covers changing the state of the device without sending, or receving. One example is setting the baudrate on a serial port.

Parameters
[in]deviceThe device you want to control. Acceptable value are listed in dn_device_t.
[in]requestThe operation to be performed on the device. The value of this parameter is device-specific.
[in]argsA pointer to the arguments needed to perform this operation. The content of this structure, or whether a structure is needed at all is detailed in device-specific documentation. Set to NULL if no arguments are needed.
[in]argLenWhen arguments are needed, specifies the length of the structure pointed to by the args parameter. Set to 0 if no arguments are needed.
Returns
DN_ERR_NONE when the operation on the device was successful.
DN_ERR_INVALID if the device specified does not exist.
DN_ERR_NOT_SUPPORTED if the device specified exists but does not support to be controlled through this function.
This function can also return device-specific error codes. These are listed in device documentation. Value will always be taken from dn_error_t.
int dn_close ( int  device)

Close a previously opened device.

Precondition
You must have opened the device before calling this function.
Postcondition
After this function returns successfully, the device is closed. You can not use the device without opening it again, unless noted differently in the device-specific documentation.

Your application should close a device when it's done using it. You must never close a device that you have not opened before.

Parameters
[in]deviceThe device you want to close. Acceptable value are listed in dn_device_t.
Returns
DN_ERR_NONE when the device was successfully closed.
DN_ERR_INVALID if the device specified does not exist.
DN_ERR_NOT_SUPPORTED if the device specified exists but does not support to be closed.
This function can also return device-specific error codes. These are listed in device documentation. Value will always be taken from dn_error_t.
int dn_read ( int  device,
char *  buf,
int  bufSize 
)

Read from a previously opened device.

Precondition
You must have opened the device before calling this function.
Postcondition
This function might change the contents of the memory pointed out by the buf parameter, but never more than the number of bytes specificed in the bufSize parameter.
Parameters
[in]deviceThe device you want to read from. Acceptable value are listed in dn_device_t.
[out]bufA pointer to a location in memory where this function will write the value. How many bytes will be written depends on the device, and is detailed in device-specific documentation.
[in]bufSizeThe maximum number of bytes this function can write to the memory location pointed to by the buf parameter. This function will never write more that this number of bytes, but might write less (see return code).
Note
All error codes are negative numbers. Getting a strictly negative return code indicates an error; getting a zero or positive number indicates the command succeeded.
Returns
The number of bytes written by this function to the memory location pointed at by the buf parameter. This can be less or equal than the value specified in the bufSize parameter. The first written byte if always written at the location pointed at by the buf parameter; bytes are written consecutively.
DN_ERR_INVALID if the device specified does not exist.
DN_ERR_NOT_SUPPORTED if the device specified exists but does not support to be read from.
This function can also return device-specific error codes. These are listed in device documentation. Value will always be taken from dn_error_t.
