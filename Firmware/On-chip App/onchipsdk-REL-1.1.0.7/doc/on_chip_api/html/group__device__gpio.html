Description

General Purpose Input/Output pin devices.

Note
Use the Device API to control this device. This page details the parameters to pass to the functions of that API.
The On-Chip Firmware Development Guide contains code snippets and sample applications, some of which use this device.

Your application can interact with the several GPIOs listed in the On-Chip Firmware Development Guide by accessing their corresponding device. To interact with a single GPIO, use its device directly (e.g. DN_GPIO_PIN_20_DEV_ID). If you want to set multiple GPIO simultaneously, you can use the DN_GPIO_OUT_PINS_DEV_ID device, as detailed in Set multiple pins simultaneously section.

The following table indicates which Device API functions apply to this device.

dn_open dn_ioctl dn_read dn_write dn_close
YES YES YES YES YES

Open the device

You need to call dn_open() to open this device before using any of the other functions. To open this device, use the dn_open() function with the following parameters:

Parameter Name Parameter Value
device The identifier of the GPIO of interest, taken from dn_device_t
args NULL
argLen 0

Control the device

Precondition
Before using this function, make sure to open the device using dn_open().

You can control a previously opened GPIO device in different ways. Use the request parameter of the dn_ioctl() function to specify which operation you want to do:

  • use DN_IOCTL_GPIO_CFG_OUTPUT to configure the pin as output. In this mode, write to the device using dn_write() to set the pin high/low.
  • use DN_IOCTL_GPIO_CFG_INPUT to configure the pin as input. In this mode, read from the device using dn_read() to return whether the pin is being driven high/low by some external device.
  • use DN_IOCTL_GPIO_ENABLE_NOTIF to enable your application to receive a notification when the state of the pin changes.
  • use DN_IOCTL_GPIO_CFG_TRISTATE to switch the pin to a tristate mode (floating).

Read from the device

Precondition
Before using the function, make sure to:
  • open the device using dn_open().
  • configure the device in input mode using dn_ioctl().

To read the state of the pin, use dn_read() with the following parameters:

Parameter Name Parameter Value
device The identifier of the GPIO of interest, taken from dn_device_t.
buf Points to a variable of type INT8U.
bufSize Size of INT8U.

After this function returns successfully, the location pointed to by the buf parameter contains:

  • 0x01 if the pin was driven high when the function was executed.
  • 0x00 if the pin was driven low when the function was executed.

Write to the device

Precondition
Before using the function, make sure to:
  • open the device using dn_open().
  • configure the device in output mode using dn_ioctl().

To set the output level of a single pin, create a INT8U variable and set that to 0x01 (resp. 0x00) if you plan on setting the pin high (resp. low).

To set the ouptut of the pins, call the dn_write() function with the following parameters.

Parameter Name Parameter Value
device The identifier of the GPIO of interest, taken from dn_device_t.
buf Points to the variable of type INT8U created earlier.
len Size of INT8U.

Set multiple pins simultaneously

To set the output level of a group of pins, use the dn_write() on special device DN_GPIO_OUT_PINS_DEV_ID. Pass a pointer to a dn_gpio_out_pins_cfg_t struct. In that struct, you can specify a bitmap for the pins to set and a separate bitmap for the pins to clear. Clearing/setting pins will then happen at the same time.

GPIO interrupts

To receive a notification when an external device drives a pin high or low:

  • create an asynchronous channel (see Channel documentation).
  • create a variable of type dn_gpio_ioctl_notif_enable_t, and populate it according to the documentation at dn_gpio_ioctl_notif_enable_t. In particular, this structure allows you to indicate the identifier of the channel you just created.
  • call the dn_ioctl() function with the following parameters:
Parameter Name Parameter Value
device The identifier of the GPIO of interest, taken from dn_device_t.
buf Points to the variable of type dn_gpio_ioctl_notif_enable_t created earlier.
len Size of dn_gpio_ioctl_notif_enable_t.

After calling this function, a notification of type dn_gpio_notif_t is triggered when one of the pins you enabled GPIO notifications on is switched to the level of interest. To receive those, your application need to regularly call dn_readAsyncMsg() (see Channel documentation). In a typical application, you can dedicate a uC/OS-II task to wait for those notifications and act accordingly.

Once a notification is triggered, further notifications are disabled until another call to dn_ioctl() re-enables notifications.

Note
The notification is level-sensitive, i.e. it does not require an edge transition to trigger. If the input pin is already at the active level when notifications are enabled, a notification will immediately be generated.
Warning
For debouncing and preventing multiple triggers, your application is responsible for inserting a delay before re-enabling notifications.

Sufficient memory must be present in the notification channels to prevent notifications from being dropped in cases where multiple pin notifications may trigger simultaneously.

Close the device

Close the GPIO by calling the dn_close() function. This will release the device and allow a (possibly different) task to open and use it.

Note
Closing a GPIO is merely "administrative", i.e. it does not change its state. That is, if the GPIO is configured as output, it will remain so after the dn_close() has been closed.
