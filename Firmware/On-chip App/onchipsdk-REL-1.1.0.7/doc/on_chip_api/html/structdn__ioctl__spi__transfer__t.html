Detailed Description

Structure to pass in the args parameter of dn_ioctl().

Warning
An extra trailing byte of value 0x00 is sent by the SPI master over the SPI bus at the end of each SPI transaction when the following combination is used:
  • dn_ioctl_spi_transfer_t::clockPhase is set to DN_SPI_CPHA_1.
  • dn_ioctl_spi_transfer_t::numSamples is larger than 1.

Field Documentation

txData

Data to be written to the SPI slave over the MOSI line.

This buffer must contain exactly dn_ioctl_spi_transfer_t::transactionLen bytes.

When dn_ioctl_spi_transfer_t::numSamples is set to 1, the data pointed to by this parameter is written exactly once over the MOSI line.

When dn_ioctl_spi_transfer_t::numSamples is set to greater than 1, the data pointed to by this parameter is written multiple times over the MOSI line.

rxData

Pointer to the memory buffer to be filled with data received from the SPI slave over the MISO line.

When dn_ioctl_spi_transfer_t::numSamples is set to 1, data received from the SPI slave will be written into this buffer.

In this case, this buffer should hold at least dn_ioctl_spi_transfer_t::transactionLen bytes.

When dn_ioctl_spi_transfer_t::numSamples is set to greater than 1, data received from the slave for each transaction will be concatenated in this buffer one after the other.

The data received over the MISO line during transaction n will be written at the next word-boundary after the data received from transaction n-1. That is, if the transaction length is set to 15 bytes, the data received during the first transaction will be written in the buffer starting a byte 0, the data received during the second transaction will be written starting at byte 15.

You need to take this alignment constraint into account when determining the number of bytes held by the received buffer.

As a first example, assume you set dn_ioctl_spi_transfer_t::transactionLen to 8 and dn_ioctl_spi_transfer_t::numSamples to 5, then your receive buffer needs to be able to hold at least 40 bytes.

As a first example, assume you set dn_ioctl_spi_transfer_t::transactionLen to 15 and dn_ioctl_spi_transfer_t::numSamples to 5, then your receive buffer needs to be able to hold at least 80 bytes. This is because each transaction occupies 16 bytes in the reception buffer: the 15 received byte an a single 0x00 padding byte.

transactionLen

Number of bytes to transmit/receive during an SPI transaction.

During a transaction, dn_ioctl_spi_transfer_t::transactionLen bytes are read from the dn_ioctl_spi_transfer_t::txData buffer and sent to the SPI slave over the MOSI line.

At the same time, dn_ioctl_spi_transfer_t::transactionLen bytes are read from the SPI slave over the MISO line and written into the dn_ioctl_spi_transfer_t::rxData buffer.

When dn_ioctl_spi_transfer_t::numSamples is set to 1, this is the length of the (single) transaction. In this case, the minimum length is 1 byte. The maximum length is 512 bytes when CPHA==0, or the value passed in the dn_spi_open_args_t::maxTransactionLenForCPHA_1 parameter when opening the device when CPHA==1.

When dn_ioctl_spi_transfer_t::numSamples is set to greater than 1, the minimum length is 1 byte. The maximum length is 32 bytes when CPHA==0, or 31 bytes when CPHA==1.

slaveSelect

Slave select line to use.

Acceptable values are:

  • DN_SPIM_SS_0n
  • DN_SPIM_SS_1n
  • DN_SPIM_SS_2n
startDelay

Delay between asserting slave select and the first clock edge.

Units are in multiples of 8 CPU clock periods (for a clock at 7.4MHz).

Acceptable values are: 0, 1, 2 and 3.

bitOrder

SPI bit order.

Acceptable values are:

  • DN_SPI_LSB_FIRST
  • DN_SPI_MSB_FIRST
clockPolarity

SPI clock polarity.

Acceptable values are:

  • DN_SPI_CPOL_1
  • DN_SPI_CPOL_0
clockPhase

SPI clock phase.

Acceptable values are:

  • DN_SPI_CPHA_1
  • DN_SPI_CPHA_0
clockDivider

SPI clock divider.

The default clock speed is 7.4MHz, i.e.

using DN_SPI_CLKDIV_4 results in an SPI clock speed of 1.85MHz.

Acceptable values are:

  • DN_SPI_CLKDIV_2
  • DN_SPI_CLKDIV_4
  • DN_SPI_CLKDIV_8
  • DN_SPI_CLKDIV_16
numSamples

Number of consecutive SPI transactions.

You can set this element to any value between 0x0001 and 0xffff.

Beware, however, that you need to allocate enough bytes in the buffer pointed to by dn_ioctl_spi_transfer_t::rxData to hold all the bytes received from the SPI slave.

See dn_ioctl_spi_transfer_t::transactionLen for more information regarding the length of the SPI transaction for different values of dn_ioctl_spi_transfer_t::numSamples.

samplePeriod

Duration between SPI transactions.

This parameter is only used when dn_ioctl_spi_transfer_t::numSamples is set to greater than 1.

The duration between the start of two transactions, in micro-seconds. Can not be strictly smaller than 100us, or strictly larger than 16383us.
