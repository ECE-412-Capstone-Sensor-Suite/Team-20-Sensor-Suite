flashinfo
Other

Read the BSP parameters from flash memory. More...

Description

Read the BSP parameters from flash memory.

Note
Use the Device API to control this device. This page details the parameters to pass to the functions of that API.
The On-Chip Firmware Development Guide contains code snippets and sample applications, some of which use this device.

You can configure the BSP parameters written to flash using the tool documented in the Eterna Board Specific Configuration Guide. This device allows your application to read those parameters.

Note
Refer to the Eterna Board Specific Configuration Guide for documentation on the different BSP parameters.

The flashinfo device allows your application to read information stored in permanent flash. The elements of information you can access are listed in the dn_bsp_param_tag_id enum.

The following table indicates which Device API functions apply to this device. Calling one of the unsupported functions on this device returns DN_ERR_NOT_SUPPORTED. Only the supported functions are detailed below.

dn_open dn_ioctl dn_read dn_write dn_close
YES NO YES NO YES

Open the device

Call dn_open() to open this device:

Parameter Name Parameter Value
device DN_FLASHINFO_DEV_ID
args NULL
argLen 0
Note
This function is provided so all devices have the same interface. This function, however, has no effect on the state of the device. This means that you can use this device without calling this function. This also means that you cannot use the return value of this function to decide whether another thread has already opened this device.

Read from the device

To read the value of one of the elements listed in dn_bsp_param_tag_id, follow these steps:

  • declare a variable type dn_bsp_param_read_t.
  • specify the element you want to access in the input.tag parameter of that variable. Use one of the values from dn_bsp_param_tag_id.

Call dn_read() with the following parameters:

Parameter Name Parameter Value
device DN_FLASHINFO_DEV_ID
buf Points to the dn_bsp_param_read_t structure.
bufSize Size of dn_bsp_param_read_t.

Make sure dn_read() returns a positive number representing the number of bytes written to the location pointed by the buf parameter.

Postcondition
This function will have changed the contents of the location pointed to by the buf parameter with the value of the requested parameter.

Close the device

Close this device by calling the dn_close() function.

Note
This function is provided so all devices have the same interface. This function, however, has no effect on the state of the device. This means that you can use use this device even after calling this function.
