Field Documentation

maxTransactionLenForCPHA_1

Maximum number of bytes in a transaction when CPHA is 1.

This parameter only applies when CPHA is 1, otherwise it must be set to zero.

It is used as a one time allocation of memory used for transactions with CPHA set to 1. Allocation of this memory must be greater than 1 byte.

Warning
We strongly recommend to keep the value of this parameter below 32 (bytes), since it determines the amount of memory being allocated.

If you close and re-open the SPI device, you must pass a number which is equal or smaller than the one passed when you first opened this device. If not, the call to dn_open() will fail and return DN_ERR_SIZE.
